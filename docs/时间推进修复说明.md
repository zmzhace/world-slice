# 时间推进修复说明

## 问题描述

用户反馈：点击"时间推进"按钮后，agents 什么都不做，没有任何行动。

## 问题原因

前端调用 `runWorldTick` 时没有传递 `panguRegistry` 参数：

```typescript
// 修复前
const nextWorld = await runWorldTick(world)
```

在 `orchestrator.ts` 中，如果没有 `panguRegistry`，会提前返回，不执行 NPC agents：

```typescript
if (!options.panguRegistry) {
  await bus.emit('after_tick', { world: baseNext })
  return baseNext  // 提前返回，NPC agents 不执行
}

// 并行执行 Pangu agents 和 NPC agents
const [panguResults, npcResults] = await Promise.all([
  options.panguRegistry.runAll(world),
  executeNpcAgents({ ...world, agents: { ...world.agents, npcs: activeNpcs } }),
])
```

## 修复方案

在前端调用 `runWorldTick` 时传递一个简单的 `panguRegistry`：

```typescript
// 修复后
const panguRegistry = {
  runAll: async () => [] // 返回空数组，让 NPC agents 自己执行
}

const nextWorld = await runWorldTick(world, { panguRegistry })
```

## Agents 行动逻辑

修复后，agents 会根据以下逻辑做出决策和行动：

### 1. 决策逻辑（makeAgentDecision）

agents 会根据自身状态和性格做出决策：

- **能量低 (< 30%)** → 休息恢复
- **压力高 (> 70%)** → 放松减压
- **有目标 + 主动性高** → 追求目标
- **共情力高 + 开放性高** → 社交互动
- **稳定性低 + 世界压力大** → 对压力做出反应
- **其他情况** → 探索世界

### 2. 行动效果（decisionToPatch）

每种行动都会影响 agent 的状态：

#### 休息 (rest)
- ✅ 能量 +20%
- ✅ 压力 -15%
- ✅ 睡眠债 -10%
- 情绪：放松

#### 放松 (relax)
- ✅ 压力 -20%
- ❌ 能量 -5%
- 情绪：平静

#### 追求目标 (pursue_goal)
- ❌ 能量 -15%
- ❌ 压力 +10%
- ✅ 专注 +10%
- 情绪：专注
- 10% 概率完成目标（移除第一个目标）
- 成功指标：power +1

#### 社交 (socialize)
- ❌ 能量 -8%
- ✅ 压力 -10%
- ✅ 关系 +0.1（与目标 agent）
- 情绪：愉快
- 成功指标：reputation +1

#### 对压力反应 (react_to_pressure)
- ❌ 压力 +15%
- ❌ 能量 -10%
- 情绪：焦虑

#### 探索 (explore)
- ❌ 能量 -10%
- ✅ 专注 +5%
- 情绪：好奇
- 成功指标：knowledge +1

#### 所有行动
- 衰老指数 +0.001（轻微衰老）

### 3. 事件生成

每个 agent 的行动都会生成一个微观事件：

```typescript
{
  id: `agent-${agent.genetics.seed}-${world.tick}`,
  kind: 'micro',
  summary: `${agent.identity.name} ${action.type} (${action.target})`,
  conflict: false
}
```

## 时间引擎集成

修复后，时间引擎也会正常工作：

1. **时间模式初始化** - 为每个 agent 分配作息时间
2. **活跃度判定** - 根据当前时间判断哪些 agents 应该活跃
3. **选择性执行** - 只有活跃的 agents 才会执行决策

例如：
- 早上 6-8 点：大部分 agents 醒来，开始活动
- 中午 12-14 点：部分 agents 休息
- 晚上 22-6 点：大部分 agents 睡眠

## 验证方法

1. 创建一个新世界并初始化（生成 5-10 个 agents）
2. 点击"⏩ +1 Tick"按钮
3. 查看"事件日志"标签页，应该能看到：
   - `🎭 Agent 行动` 事件
   - 每个 agent 的具体行动（如"张三 pursue_goal"）
4. 查看"女娲造人"标签页，agents 的状态应该有变化：
   - 能量、压力、专注等数值变化
   - 情绪状态更新
   - 关系网络变化
5. 点击"▶️ 自动推进"按钮，设置推进 10 个 ticks
   - agents 会连续执行 10 次决策
   - 可以看到状态的累积变化

## 涌现式叙事

随着 agents 的互动，叙事系统会自动识别模式：

- **冲突叙事** - agents 之间的对立和竞争
- **合作叙事** - agents 之间的协作和互助
- **探索叙事** - agents 探索世界的过程
- **成长叙事** - agents 追求目标的历程
- **关系叙事** - agents 之间关系的发展

这些叙事会在"📖 涌现叙事"标签页中显示。
