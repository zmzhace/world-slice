# P0 优先级功能实现总结

## 实现日期
2026-03-14

## 实现内容

### 1. 推荐系统 (Recommendation System)
**文件**: `src/engine/recommendation-system.ts`

**功能**:
- 基于兴趣的推荐：计算 agent 兴趣向量与事件的相似度
- 基于热度的推荐：参考 Reddit 热度算法，考虑时间衰减和事件重要性
- 基于社交网络的推荐：推荐关注的人的相关动态
- 混合推荐：综合三种推荐策略，可配置权重

**配置参数**:
```typescript
{
  topK: 10,              // 推荐数量
  interestWeight: 0.4,   // 兴趣权重
  hotWeight: 0.3,        // 热度权重
  socialWeight: 0.2,     // 社交权重
  recencyWeight: 0.1     // 时间新鲜度权重
}
```

**API**:
- `recommendEvents(agent, world)`: 为 agent 推荐事件
- `createRecommendationSystem(config)`: 创建推荐系统实例

### 2. 时间引擎 (Time Engine)
**文件**: `src/engine/time-engine.ts`

**功能**:
- 24 维活跃度概率：每小时的活跃概率 [0-1]
- 基于性格生成作息模式：
  - 夜猫子 (openness > 0.6 && agency > 0.5): 凌晨 2 点睡，上午 10 点起
  - 早起者 (stability > 0.6 && agency > 0.6): 晚上 10 点睡，早上 6 点起
  - 普通作息: 晚上 11 点睡，早上 7 点起
- 动态激活 agents：根据当前时间和活跃度概率决定是否激活
- 活跃度曲线生成：考虑起床时间、睡觉时间、午餐晚餐时间

**数据结构**:
```typescript
type TimePattern = {
  hourly_activity: number[]  // 24 维活跃概率
  timezone_offset: number    // 时区偏移
  sleep_schedule: {
    typical_sleep_hour: number
    typical_wake_hour: number
  }
}
```

**API**:
- `shouldActivateAgent(agent, currentHour)`: 判断 agent 是否应该激活
- `getActiveAgents(agents, world)`: 获取当前时间应该激活的 agents
- `generateTimePattern(agent)`: 为 agent 生成时间模式
- `initializeTimePatterns(agents)`: 为所有 agents 初始化时间模式
- `getActivityStats(world)`: 获取活跃度统计

### 3. 知识图谱 (Knowledge Graph)
**文件**: `src/engine/knowledge-graph.ts`

**功能**:
- 节点类型：agent, location, organization, event, concept, plot
- 关系类型：knows, likes, dislikes, works_for, located_in, participates_in, caused_by, related_to, protagonist_of, antagonist_of, supports
- 图查询：
  - 获取节点的邻居
  - 查询 agent 相关的所有信息（深度优先搜索）
  - 查找两个节点之间的最短路径
- 自动构建：从世界状态构建知识图谱
- 导入导出：支持 JSON 格式

**数据结构**:
```typescript
type KnowledgeNode = {
  id: string
  type: NodeType
  label: string
  properties: Record<string, any>
  embedding?: number[]  // 用于语义搜索（未来可接入 OpenAI Embeddings）
  created_at: number
  updated_at: number
}

type KnowledgeEdge = {
  id: string
  source: string
  target: string
  relation: RelationType
  weight: number
  properties: Record<string, any>
  created_at: number
  updated_at: number
}
```

**API**:
- `addNode(node)`: 添加节点
- `addEdge(edge)`: 添加边
- `getNode(id)`: 获取节点
- `getNodesByType(type)`: 获取某类型的所有节点
- `getNeighbors(nodeId)`: 获取节点的所有邻居
- `getAgentContext(agentId, depth)`: 查询与 agent 相关的所有信息
- `findShortestPath(sourceId, targetId)`: 查找最短路径
- `getStats()`: 获取图谱统计信息
- `buildFromWorld(world)`: 从世界状态构建知识图谱
- `toJSON()` / `fromJSON()`: 导入导出

### 4. 系统统计面板 (System Stats Panel)
**文件**: `src/components/panel/system-stats-panel.tsx`

**功能**:
- 时间引擎统计：当前时间、活跃 agents、活跃率、休眠 agents、活跃度进度条
- 知识图谱统计：总节点数、总边数、平均度数、节点类型分布
- 推荐系统状态：运行状态、可推荐事件数
- 系统性能：当前 Tick、总事件数、活跃剧情数

### 5. Orchestrator 集成
**文件**: `src/engine/orchestrator.ts`

**集成内容**:
- 时间引擎：
  - 初始化所有 agents 的时间模式
  - 获取当前时间应该激活的 agents
  - 只对激活的 agents 执行更新和行动
  - 输出活跃度统计日志
- 知识图谱：
  - 每个 tick 重建知识图谱
  - 输出图谱统计日志（节点数、边数）
- 推荐系统：已创建实例（待后续集成到 agent 决策中）
